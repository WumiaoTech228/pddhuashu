02. 数据备份与安全警告
C盘数据备份警示
重要提示：重装系统会清空C盘的所有数据，包括桌面文件、文档、下载目录以及微信/QQ的聊天记录。请务必将重要数据备份到其他分区中！
多硬盘/多分区安全告知
请确认您的电脑上除了C盘外是否还有其他盘（如D盘、E盘）？如果有，重装系统默认只格式化C盘，其他盘的文件均会安全保留，不受影响。
浏览器书签与账号备份提示
如果您有在浏览器中保存的重要书签、账号密码或行业软件本地数据，请提前导出备份。系统重装后，原有软件需重新安装，浏览器数据可登录账号进行云同步。
磁盘 BitLocker 加密警告
重要安全提示：重装前请确认您的 C 盘或 D 盘等分区是否开启了 BitLocker 磁盘加密（如果开启，盘符上会有个锁的图标）。若开启了，请务必先进入控制面板将其彻底解密关闭，或者导出保存好您的恢复密钥，以防重装后其他盘被死锁无法读取。

07. 重启引导与自动安装
重启进入自动重装环境
重装软件环境已经配置完毕，现在需要重启电脑。重启后电脑会自动显示引导菜单并进入重装软件的PE自动安装环境，您无需敲击任何按键，网络及声音自动部署，耐心等待其自动执行格式化和写入即可。
重启进不去BIOS/启动菜单解决方法
如果重启后没有自动进入重装环境，而是直接回到了原系统或提示引导失败，说明您的系统开启了引导保护。请先手动关机，开机时连击 Delete 或 F2 进入 BIOS，在安全或启动选项中检查引导顺序。
安全启动报错解决方法
如果重启进入重装软件环境时屏幕报错或提示安全启动违规（Secure Boot Violation），需要重新开机进BIOS，在安全设置（Security）里将Secure Boot（安全启动）设置为Disabled（关闭），保存重启后再试。
老旧电脑 Legacy 与 UEFI 引导模式切换
如果重启后找不到重装软件的引导项，可能是老旧主板未开启 UEFI 兼容。请进入 BIOS 设置，在 Boot 或 Startup 菜单中，将 Boot Mode 从 Legacy 修改为 UEFI（或者将 CSM 兼容支持模块开启），保存并重启后再试。
08. 安装中遇到特殊问题解决
看不到电脑硬盘（Intel VMD限制）
如果重装界面找不到任何磁盘，这是因为您的电脑是Intel 11代及以上的处理器，默认开启了Intel VMD硬盘技术。我们需要远程为您加载相应的IRST驱动，或者需要您进入BIOS关闭VMD设置。
引导格式不符（GPT与MBR硬盘转换）
如果重装软件提示无法安装到当前磁盘，因为硬盘分区表是 MBR 或 GPT 格式不符，这是因为您的系统分区表类型与当前的启动模式冲突。工程师需要将该磁盘重新初始化并转换分区表格式，请确认。
重装后其他盘符（D盘/E盘）不见了或显示脱机
如果进入桌面后发现 D 盘或 E 盘等其他分区不见了，请右键点击桌面“此电脑”选择“管理”，进入“磁盘管理”。如果看到其他硬盘状态为“脱机”，只需在其上点击右键选择“联机”；如果显示未分配盘符，右键为其“更改驱动器号和路径”分配盘符即可恢复。

针对您的网络访问异常或游戏高延迟丢包，我们将为您进行网卡驱动重置、清空 DNS 缓存并配置最优公共 DNS（如阿里或腾讯DNS），并修复 LSP 套接字以恢复正常网络。

Ubuntu Desktop (图形化) 与 Server (命令行) 安装教程
【Ubuntu 安装指南】 一、Ubuntu Desktop (图形化桌面版) 1. 引导启动：使用 Ventoy 启动 U 盘，选择 Ubuntu Desktop 镜像，选择 "Try or Install Ubuntu" 进入 Live 桌面。 2. 启动安装：双击桌面的 "Install Ubuntu" 图标，选择语言（中文简体）。 3. 键盘与网络：选择 "Chinese" 键盘布局，连接网络（建议先断开网络，否则联网会自动下载更新导致安装极慢）。 4. 更新与软件：选择 "正常安装"（含浏览器、办公软件和媒体播放器）或 "最小安装"。勾选 "安装第三方软件和硬件驱动"（可自动解决 Wi-Fi 和部分显卡驱动）。 5. 安装类型与分区： - 全盘安装：清除整个磁盘并安装 Ubuntu（会抹除所有数据）。 - 双系统/自定义分区：选择 "其他选项" (Something else)。 - 在空闲空间上点击 "+"，创建 EFI 系统分区 (大小建议 500MB，分区类型为主分区，用于 "EFI 系统分区")。 - 创建 Swap 交换空间 (大小建议等同物理内存，分区类型为逻辑分区，用于 "交换空间")。 - 创建根分区 (剩余所有空间，分区类型为主分区，用于 "Ext4 日志文件系统"，挂载点为 "/")。 6. 设置时区与用户：时区选择 Shanghai，输入用户名、计算机名 and 登录密码，选择登录方式。 7. 写入安装：核对分区信息，点击 "现在安装"，等待进度条结束，提示重启时点击 "现在重启" 并拔出 U 盘。 二、Ubuntu Server (命令行服务器版) 1. 引导启动：启动 U 盘选择 Ubuntu Server 镜像，加载内核。 2. 基础语言：选择安装过程语言（默认 English，建议保持英文以防控制台乱码）。 3. 键盘配置：默认 Layout/Variant 选择 "Chinese / Chinese"。 4. 选项配置：选择 "Ubuntu Server" (常规安装) 或 "Ubuntu Server (minimized)" (最小化精简安装)。 5. 网络配置：检测网卡，默认通过 DHCP 获取 IP。如需静态 IP，在此步骤手动编辑 Subnet, Address, Gateway, DNS。 6. 代理与源地址：Proxy 留空，镜像源 (Configure Ubuntu archive mirror) 更改为国内高速源，例如: https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ 7. 磁盘分区： - 默认勾选 "Use an entire disk" 且勾选 "Set up this disk as an LVM group"（使用 LVM 逻辑卷，便于后期在线扩容）。 - 手动分区选择 "Custom storage layout"： - 创建 biosgrub (1MB，若为旧式 MBR 引导) 或 /boot/efi (500MB，文件系统 fat32，挂载为 EFI系统分区)。 - 创建 /boot 分区 (1GB，Ext4)。 - 创建 LVM 逻辑卷并分配根分区 "/" (Ext4) 和 swap。 8. 身份设置：输入姓名、服务器名 (hostname)、用户名和高强度密码。 9. SSH服务：务必勾选 "Install OpenSSH server"（开启 SSH 远程连接以便远程运维），导入认证 Key 可跳过。 10. 内置软件：可选安装常见服务（如 Docker, MicroK8s, Nextcloud 等），直接点击 Done 忽略。 11. 完成重启：系统自动写入，当底部显示 "Reboot Now" 时，按回车重启并拔出 U 盘。

【Arch Linux 安装指南】 Arch Linux 遵循简洁原则，默认不提供图形安装器。 一、archinstall 脚本自动化安装 (推荐，含桌面/命令行) 1. 引导启动：U 盘引导进入 Arch 命令行 Live 环境，确保网络畅通（台式机插网线自动联网，笔记本可输入 `iwctl` 命令连接 Wi-Fi）。 2. 启动脚本：在终端输入命令 `archinstall` 并按回车。 3. 交互式菜单配置： - "Language"：选择系统语言（建议保持 English 以防控制台乱码）。 - "Mirrors" -> "Mirror region"：选择 "China" 以激活国内镜像站。 - "Disk configuration"：选择 "Layout" -> 选择目标硬盘 -> 选择 "Wipe all selected drives" (全盘清空并自动分区) -> 推荐文件系统选择 "btrfs" 或 "ext4"。 - "Loader"：选择默认的 "systemd-boot" 或 "grub"。 - "Profile"： - 若要安装图形桌面：选择 "Type" 为 "desktop" -> 选择桌面环境（如 "kde"、"gnome" 或 "xfce"） -> 选择显卡驱动（Intel/AMD 选择开源驱动，NVIDIA 选择 "nvidia" 驱动）。 - 若要安装精简服务器：选择 "Type" 为 "minimal"，不带图形环境。 - "Audio"：推荐选择 "pipewire" (现代化音频驱动)。 - "Kernels"：选择 "linux" (稳定内核) 或 "linux-lts" (长期支持内核)。 - "Create user"：创建一个普通管理员账号，输入用户名和密码，并勾选 "Should this user be a superuser" (赋予 sudo 权限)。设置 Root 超级密码。 - "Network configuration"：选择 "Use NetworkManager" (非常重要，否则开机后无法联网)。 - "Timezone"：设置为 "Asia/Shanghai"。 4. 执行安装：点击菜单底部的 "Install"，系统开始下载并安装软件包。完成后会询问是否进入 chroot 环境，选择 "no"，输入 `reboot` 重启并拔出 U 盘。 二、传统命令行手动安装 (纯命令模式) 1. 分区磁盘：使用 `cfdisk /dev/sda` 分区（EFI分区 500M，Swap分区 8G，根分区剩余）。格式化分区：`mkfs.vfat -F32 /dev/sda1`，`mkswap /dev/sda2 && swapon /dev/sda2`，`mkfs.ext4 /dev/sda3`。 2. 挂载分区：`mount /dev/sda3 /mnt`，`mkdir -p /mnt/boot`，`mount /dev/sda1 /mnt/boot`。 3. 安装基础系统：修改 `/etc/pacman.d/mirrorlist` 将中国源置顶，执行 `pacstrap -K /mnt base linux linux-firmware base-devel NetworkManager`。 4. 配置系统：生成挂载表 `genfstab -U /mnt >> /mnt/etc/fstab`。进入新系统 `arch-chroot /mnt`。 5. 系统初始化：设置时区 `ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime && hwclock --systohc`。配置本地化字符集，在 `/etc/locale.gen` 中取消注释 `en_US.UTF-8` 和 `zh_CN.UTF-8` 并运行 `locale-gen`。设置主机名 `echo "myarch" > /etc/hostname`。 6. 用户与引导：运行 `passwd` 设置 root 密码。安装 GRUB 引导：`pacman -S grub efibootmgr`，运行 `grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB` 并生成配置文件 `grub-mkconfig -o /boot/grub/grub.cfg`。 7. 启动服务：`systemctl enable NetworkManager`。输入 `exit` 退出 chroot，输入 `umount -R /mnt`，然后 `reboot` 重启。

【CentOS / RHEL 安装指南】 CentOS 与 RHEL 采用相同的 Anaconda 安装框架， RHEL 需要开发者账号或商业订阅。 一、图形化桌面安装 (GUI) 1. 引导启动：U 盘启动选择 "Install CentOS Stream" 或 "Install Red Hat Enterprise Linux"。 2. 语言配置：选择 "中文" -> "简体中文 (中国)"。 3. 软件选择： - 点击 "软件选择" (Software Selection)。 - 在左侧选择 "带 GUI 的服务器" (Server with GUI) 或 "工作站" (Workstation)。在右侧勾选附加包。 4. 磁盘分区： - 点击 "安装目的地" (Installation Destination)。 - 选中目标磁盘，选择 "自动" 分区方案即可。RHEL 系列默认会配置 XFS 文件系统与 LVM 卷管理。 5. 账户管理： - 设置 Root 用户密码。 - 创建普通用户并勾选 "将此用户作为管理员"。 6. 开启网络：点击 "网络和主机名"，确保右上角网卡开关置为 ON 状态。 7. 开始安装：点击 "开始安装"。安装完毕后点击 "重启系统"。如果是 RHEL 系统，首次进入桌面后需在终端运行 `subscription-manager register` 注册您的红帽账号以激活软件源更新。 二、极简服务器安装 (CLI Minimal) 1. 语言与地区选择同上，设为 "简体中文"。 2. 软件选择：点击 "软件选择"，选中 "最小安装" (Minimal Install)，取消所有桌面环境，以确保最高的服务器执行效能与最小的安全风险。 3. 分区目的地：选择 "自动" 存储配置方案。 4. 激活网卡：进入 "网络和主机名" 配置，开启网卡开关，使其开机自动通过 DHCP 联网。 5. 密码配置：配置 Root 强密码。 6. 开始执行：点击 "开始安装"。等待安装完毕后重启并拔掉 U 盘，开机即进入企业级命令行终端。
Oracle Linux & Anolis OS (龙蜥) 数据库与云原生系统安装教程
【Oracle Linux / Anolis OS 安装指南】 这两个发行版为兼容红帽 RHEL 的企业级系统，分别针对数据库和云原生场景进行了优化。 一、图形化桌面安装 (GUI) 1. 引导启动：U 盘引导启动，在菜单中选择 "Install Oracle Linux" 或 "Install Anolis OS"。 2. 语言选择：选择 "中文" -> "简体中文 (中国)"。 3. 软件选择： - 点击 "软件选择" (Software Selection)。 - 在左侧大类中选择 "带 GUI 的服务器" (Server with GUI) 或 "工作站" (Workstation)。 - Oracle Linux 用户建议勾选 "Oracle 数据库先决条件" 等特定包组，以自动配置系统参数（如 sysctl.conf），简化 Oracle 数据库后续的安装过程。 4. 分区与存储：选择目标硬盘，存储配置选择 "自动" (推荐采用默认 openings LVM + XFS 分区布局)。 5. 网络配置：点击 "网络和主机名"，开启网卡开关，并配置主机名以确保网络畅通。 6. 账户设置：设置 Root 管理员密码并创建一个普通管理员账号。 7. 执行写入：确认所有警告消除后，点击 "开始安装" (Begin Installation)，安装完成后重启并拔出 U 盘。 二、极简服务器安装 (CLI Minimal) 1. 前期引导与语言配置同上，选择 "简体中文" 语言。 2. 软件选择： - 点击 "软件选择"，在大类中选择 "最小安装" (Minimal Install)，不带任何图形界面。 - Oracle Linux 在最小安装下也可以在右侧勾选 "Oracle Base System" 等包，以便在命令行环境下运行 Oracle 数据库。 3. 分区与网络：选择目标硬盘为 "自动" 分区方案。在 "网络和主机名" 中开启网卡。 4. 账户配置：设置 Root 超级密码与管理员普通账号。 5. 开始安装：点击 "开始安装" 并等待系统写入重启。开机后，可以使用系统默认的高性能内核进行开发和生产环境部署。

17. Office 办公软件安装
IMG 离线镜像与在线安装的区别
离线安装包（如本站提供的 .img 镜像文件）体积较大（通常为 3-5 GB），内部已包含全部组件数据，在没有网络连接或网络极差的状况下也能本地直接挂载运行 setup.exe 安装，绝无卡死风险。 而在线安装包（Web Installer）仅有几 MB 大小，运行后需实时连接微软服务器在线下载组件，在网络较差时经常会出现“Office 正在后台下载”、“正在准备”卡在 0% 或报错 30015 等情况导致失败。
C2R 即点即用与传统 MSI 安装的区别
C2R（Click-to-Run，即点即用）是微软近年来主推的流式安装技术。在 2019/2021/2024 等新版中，微软默认强制采用 C2R 格式，不再提供传统的图形自定义界面，双击安装便会自动部署全套组件（Word/Excel/PPT/Outlook 等全家桶），如需剔除不常用组件，必须使用 XML 配置文件或第三方工具（如 Office Tool Plus）进行部署定制。 而传统的 MSI 安装方式则用于 Office 2016 及更早版本，支持在安装向导中点击“自定义安装”，自由勾选需要或剔除不需要的软件组件（如只保留 Word、Excel、PPT）。
微软 ODT 部署工具命令行配置使用
如需自定义安装 Office 大量授权版（LTSC 2021/2024）或实现组件精细化配置，建议使用微软官方的 Office 部署工具 (ODT)： 1. 下载 ODT 工具并在本地空目录解压，得到 setup.exe 和示例的 configuration.xml。 2. 使用记事本打开并编辑 configuration.xml，在 <Add> 节点中配置具体的 Channel、Product ID（如 ProPlus2024Volume）、Language 语言，以及通过 <ExcludeApp> 剔除不需要的组件（如 OneDrive）。 3. 以管理员身份运行 CMD 命令提示符，切换至 setup.exe 目录。 4. 运行配置安装命令：setup.exe /configure configuration.xml 系统将依据 XML 配置自动静默安装。
推荐使用 Office Tool Plus 辅助部署
如需图形化、无代码地下载并部署 Office，本站强烈推荐使用开源绿色的 Office Tool Plus 辅助工具： 1. 点击直链下载 Office Tool Plus (OTP) 并解压运行（无需安装）。 2. 进入“部署”板块，您可以直观勾选想要安装的 Office 版本（零售/批量版，支持 365/2024/2021/2016/2013 等）、选择架构（64位/32位）以及希望安装的独立应用（如仅安装 Word/Excel/PPT，取消勾选 Outlook/Access/OneNote）。 3. 部署安装通道选择“即点即用 (C2R)”，安装模块保持默认，点击“开始部署”即可自动下载并安装完毕，安全纯净无劫持。
